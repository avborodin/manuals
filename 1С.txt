Администрирование
==============================



C:\Program Files (x86)\1cv8t\common\
1cestartt.exe - главный стартер платформы (вне версионный) "1cestart"

....\bin\1cv8st.exe это тонкий клиент для операторов, один режим: 1С:Предприятие 
....\bin\1cv8t.exe это толстый клиент для админов, разработчиков и настройщиков, 
два режимы работы: 1С:Предприятие и Конфигуратор 


1. Запускает стартер самой свежей версии платормы "1cv8s"
C:\Program Files (x86)\1cv8t\8.3.20.1789\bin\1cv8st.exe

2. Установщик платформы (только в ком. платформе)


======
"Информационная база" - файл "1Cv8.1CD"
Для работы с данными необходимо указать их размещение и назначение

"Информационная база" = Конфигурация + Данные 

Конфигурация - инструкции для платформы
Данные - учетные и технические


======
прикладное решение = платформа + Информационная база

=================================
Размещение технической информации в платформе 1С:Предприятие 8
--
1. Список ИБ (ibases.v8i)
C:\Users\<UserName>\AppData\Roaming\1C\1CEStartt

Для перехода к папке C:\Users\<UserName>\AppData\Roaming
можно в диалоге запуска (Win+R) ввести: %AppData%

1C\1CEStartt - для учебной платформы
1C\1CEStart - для ком. платфррмы

Определение списка информационных баз для пользователя задается размещением в папке профиля пользователя

Назначение:
- Возможность скопировать список баз на другой компьютер или другому пользователю
- Содержимое файла дает нам информацию о размещении тех. данных
клиентских приложений 

Важный параметр для базы:ID
Значение данного параметра определяет размещение тех. данных
в папке: C:\Users\User1C\AppData\Roaming\1C\1cv8t\<ID>

например,
C:\Users\User1C\AppData\Roaming\1C\1cv8t\d132fc72-098f-4348-ad9a-71552bc7f4c5

Каталог соответствует регистрации базы с ID (d132fc72-098f-4348-ad9a-71552bc7f4c5)

1cv8.pfl - параметры работы толстого клиента
1cv8c.pfl- параметры работы тонкого клиента

Есть параметры общие (единые) для всех баз:
C:\Users\User1C\AppData\Roaming\1C\1cv8t
1cv8.pfl - параметры работы толстого клиента
1cv8c.pfl- параметры работы тонкого клиента
1cv8cmn.pfl - параметры без привязки к виду клиента
1cv8strt.pfl - параметры окна запуска

===

Папку  C:\Users\User1C\AppData\Roaming\1C не удаляем
Там хранятся настройки программы (платформы)

===========================================
Есть временные данные платформы:
КЭШ - C:\Users\User1C\AppData\Local\1C
Временные файлы - C:\Users\User1C\AppData\Local\Temp

- Можно удалять после завершения работы пользователя в ОС
Главное:
Эти папки(файлы) желательно размещать на быстрых дисках

Перенос данных папок на скоростные диски увеличивает производительность клиентских приложений и сервера 1С

На Windows
можно использовать ссылки на каталог

Переносим данные каталога хранения кэш в отдельный каталог (диск)
Начальное размещение: C:\Users\User1C\AppData\Local\1C
Нужно перенести его  по адресу:
c:\1c\data\Local1C
Но, система должна обращать к папке по старому адресу
Лог. адрес(C:\Users\User1C\AppData\Local\1C)
Физ.адрес(c:\1c\data\Local1C)


Пример реализации:
1) Создаем физ. адрес: c:\1c\data\Local1C
2) Переносим в новую папку все данные из каталога C:\Users\User1C\AppData\Local\1C
3) Удаляем папку: C:\Users\User1C\AppData\Local\1C
4) mklink /J C:\Users\User1C\AppData\Local\1C c:\1c\data\Local1C

Получили по адресу: C:\Users\User1C\AppData\Local\1C - ссылку на папку
Этот адрес будет использовать платформа:

Но, физически файлы будут располагаться в другом месте
c:\1c\data\Local1C

Ссылка будет работать на все учетные записи

Даже если стоит ssd на весь комп, желательно вывести в виде папки с -отдельным разделом диска-
отдельный раздел диска уменьшает общую раздефрагментацию

Скорость - диски ssd (время жизни меньше чем у hdd)
Надежность хранения - диски hdd

====

Влияние временных файлов на производительность:
Все клиентские приложения, все сервер. приложения (1С) активно пишут временные файлы на диск

Временные файлы на стороне сервера: внешние обработки и отчеты (инструменты из меню тех. специалиста)

=================

C:\1C\data\InfoBases\sample\ - папка с базой
1Cv8Log - папка с журналом регистрации оперативным (что делали пользователи)
Оперативный предназначен для фиксации изменений (скорость важна для записи)

Для анализа нужно выгружать из оперативного в архивный журнал регистрации:
====
При работе пользователей активно пишется информация в ЖР и при чтении получим тормоза

Читаем из отдельного ЖР (архивного) и анализируем

ЖР ведется для каждой базы отдельно
Если используется сервер, то ЖР хранится на стороне сервера 1С, т.к. нет каталога базы 1С

====

Если диск с ЖР медленный, производительность работы с определенной базой может упасть

=====

Сокращение ЖР можно автоматизировать на языке 1С и поместить на исполнение по расписанию
метод 1С: СократитьЖурналРегистрации

Как создать общий список информационных баз для всех пользователей компьютера

Есть каталог для определенного пользователя:
%AppData%\Roaming\1C\1CEStartt

Общий каталог для стартера 1С с настройками:
C:\ProgramData\1C\1CEStartt
1cestart.cfg

В данном файле можно прописать общий для всех пользователей список ИБ:
CommonInfoBases=C:\ProgramData\1C\1CEStartt\ibases.v8i

В данном варианте в настройке окна запуска будет показываться путь к файлу со списком ИБ
(нужно убрать отображение)

=======
Определяем список ИБ единый и автоподключаемый для нескольких компьютеров
(путь к файлу будет скрытым)

Размещаем ibases.v8i на сетевом ресурсе
В нашем примере сделаем имитацию сетевого ресурса в виде папки: c:\1c\data\iblist
В папку c:\1c\data\iblist переносим файл списка ИБ из C:\ProgramData\1C\1CEStartt
Копируем файл настройки из C:\ProgramData\1C\1CEStartt\1cestart.cfg в папку c:\1c\data\iblist
Переименовываем файл c:\1c\data\iblist\1cestart.cfg в 1cescmn.cfg
1cescmn.cfg - общий файл настроек для стартера 1С (как правило, используется на сетевом ресурсе)
В файле оставляем параметр и путь указываем новый:
CommonInfoBases=c:\1c\data\iblist\ibases.v8i
Остальное удаляем

На файл общих настроек нужно указать путь в локальном файле настроек
C:\ProgramData\1C\1CEStartt\1cestart.cfg

Удаляем: CommonInfoBases=C:\ProgramData\1C\1CEStartt\ibases.v8i
Добавляем: CommonCfgLocation=C:\1C\data\iblist\1cescmn.cfg
